## 2025-12-19

1. 📂 查 (Read/Load) —— “看看都有啥”<br>用 Python 的 os.listdir() 或 glob 模块，扫描存档文件夹。<br>找到所有 .json 文件。<br>进阶：读取每个文件的 modified time（修改时间），按时间排序，显示“最新存档”。<br>作用：让用户看到他的“战绩列表”。
2. ✏️ 改 (Update/Modify) —— “改个名字”<br>读取旧文件的数据。<br>在 Python 字典里，修改 data['name'] = "决战紫禁之巅"。<br>写入回同一个文件（覆盖）。<br>注意：本质上是“先读后写”，但文件名不变。
3. 🗑️ 删 (Delete/Remove) —— “手滑了，重来”<br>调用 os.remove('path/to/save1.json')。<br>交互：商业软件一定要加一个弹窗确认：“确定要删除吗？此操作不可恢复！”（这是基本的用户体验）。<br>作用：释放磁盘空间，让用户清爽。
4. ➕ 增 (Create/Add) —— “新开始”<br>生成一个唯一的文件名（比如 save_20251222_1900.json 或者 slot_1.json）。<br>把初始数据（字典） dump 进去。<br>注意：要处理“存档槽位已满”的情况（比如最多只允许存 10 个）。<br>核心双子星：dump vs load
5. 这是你今天的左右手，必须分清。<br>json.dump(data, file)：存。 把 Python 里的字典/列表，变成 JSON 字符串，塞进文件里。<br>json.load(file)：取。 把文件里的 JSON 字符串，变回 Python 的字典/列表。<br>口诀：Python -> 文件 是 dump（倾倒数据）；文件 -> Python 是 load（加载数据）。
2. 文件操作模式：'w' vs 'r' vs 'a'<br>这是你开文件的钥匙。<br>'w' (Write)：写入模式。 打开即清空！不管原来有什么，全删了，重新写。存档时用这个。<br>'r' (Read)：读取模式。 只读不写，文件必须得存在，不然报错。<br>'a' (Append)：追加模式。 在文件末尾加内容，不删前面的。虽然 JSON 一般不用这个（因为 JSON 格式要求严格），但你要知道有这把钥匙。
3. 异常处理：try...except<br>这是你程序的防弹衣。<br>场景：你想读一个存档文件，结果文件被删了，或者格式坏了。<br>作用：如果没有 try...except，程序直接崩给你看（报红）。<br>逻辑：“try（尝试）去读，如果（except）报错了（比如文件不存在），就执行这段备用代码（比如创建一个新文件）。”<br>结果：程序稳如老狗，用户体验满分。
4. 数据类型转换<br>这是你和计算机沟通的翻译官。<br>Python 认识：dict, list, str, int, True/False。<br>JSON 认识：对象({}), 数组([]), 字符串, 数字, true/false。<br>注意点：Python 里的 True 存进 JSON 变成 true，读出来又变回 True。它俩是“异父异母的双胞胎”，JSON 能看懂 Python 的“方言”，但存下来是标准的“普通话”。
